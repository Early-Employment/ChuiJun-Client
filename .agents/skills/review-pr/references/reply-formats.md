# GitHub 답글 형식

Step 5에서 인라인 답글을 게시할 때 사용하는 템플릿.
`comment_id`는 반드시 인용하여 쉘 인젝션을 방지한다.
모든 답글은 한국어로 작성한다.

답글 본문 맨 앞에 리뷰어 멘션을 붙인다 (`@<reviewer>`).
예: `gemini-code-assist[bot]` → `@gemini-code-assist`, 사람 리뷰어 → `@<username>`

## VALID — 수정 성공

```
@<reviewer> <abc1234> 에서 반영했습니다. (근거: <출처>)
```

## VALID — 수정 실패

```
@<reviewer> 지적 사항이 타당합니다. 직접 수정이 필요하여 별도 처리하겠습니다.
```

## INVALID

```
@<reviewer> 해당 지적은 이 프로젝트의 컨벤션과 다릅니다.
근거: <출처> — "<규칙 인용>"
현재 코드가 규칙을 올바르게 따르고 있어 변경하지 않겠습니다.
```

## PARTIAL — 수락

```
@<reviewer> 부분적으로 타당하다고 판단하여 <abc1234> 에서 반영했습니다.
```

## PARTIAL — 거절

```
@<reviewer> 검토 결과 이 방향으로는 적용하지 않기로 결정했습니다.
```

## PARTIAL — 보류

```
@<reviewer> 검토 중입니다. 추후 답변드리겠습니다.
```
