# 프로젝트 계획 수립

## Architecture

FSD 아키텍쳐를 사용하며, 계층을 위반한 경우 경고를 띄운다.

## tech stack

style: tailwind css
language: typescript
framework: 현재 프로젝트 요구사항에 따라 추천 ㄱ
signin: [data gsm](https://docs.datagsm.kr/) - client, secret 발급 예정
상태 관리 & 데이터 패칭: axios & tanstack query
database & server: 별도의 백엔드 서버가 존재한다.
format: eslint, prettier

### query key

query Options로 Query Key Factory 패턴을 사용한다.

### Optimisic Update

좋아요 등의 버튼에는 낙관적 업데이트를 사용한다.

## components

### error & loading

에러와 로딩 상태는 컴포넌트 내에서 처리하며 Suspense, Error Boundary를 적절히 사용한다.
사용 예시는 ChildComponent.Error = ~~~ 이다.

로딩 상태는 원형 로딩과 스켈레톤 UI를 적절히 사용하며, 로딩이 굳이 필요하지 않은 부분이나 레이아웃 변화가 작은 부분에는 오히려 독이되니 사용하지 않는다.
